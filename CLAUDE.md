# Правила работы с проектом

## Архитектура

- Каждая страница — один самодостаточный HTML-файл (стили внутри `<style>`, скрипты внутри `<script>`)
- Никаких внешних библиотек (React, Bootstrap, jQuery и т.д.)
- Допустимо подключение Google Fonts через `<link>`
- Фото и изображения хранятся рядом с HTML-файлами в корне проекта

## Дизайн

**Палитра адаптируется под каждый проект.** Для личных страниц (визитка, портфолио) — тёмная тема с бордовым и золотом. Для клиентских лендингов — цвета подбираются под нишу и бренд клиента. Единое остаётся: структура CSS-переменных, типографика и принципы вёрстки.

**Базовая палитра для личных страниц** (использовать CSS-переменные через `:root`):
- Фон: `#0C0806` / `#0D0A07`
- Карточки/слои: `#161009` / `#1E1410`
- Акцент красный: `#9B1C1C` (кнопки, выделения)
- Акцент золото: `#C8884A` / `#E0A060` (заголовки секций, цены)
- Текст основной: `#F5EDE0`
- Текст второстепенный: `#9A8070`
- Граница: `rgba(200,136,74,.18)`

**Типографика:**
- Заголовки: `Cormorant Garamond` — элегантная, с курсивом для акцентов
- Основной текст: `Montserrat` — чистый, современный
- Мелкие подписи и теги: Montserrat, `letter-spacing: 2-4px`, `text-transform: uppercase`

**Визуальный стиль:**
- Тёмная тема, тёплая палитра (ощущение уюта, качества, дороговизны)
- Плавные анимации появления: `fadeUp` при загрузке страницы
- Frosted glass на бейджах: `backdrop-filter: blur()`
- Радиальные градиенты для фонового свечения (не плоский чёрный)
- Hover-эффекты: `transform: translateY(-2px)` + мягкая тень

## Адаптивность

- Вёрстка должна корректно выглядеть на экранах от 360px
- Использовать `grid` с `auto-fit / minmax()` вместо фиксированных колонок
- На мобильных скрывать навигационные ссылки, оставлять только логотип и кнопку
- Размеры шрифтов заголовков через `clamp()` — масштабируются автоматически
- Кнопки и ссылки — минимум 44px по высоте для удобного нажатия пальцем

## Стиль текста

- Простой и дружеский язык, без канцелярита
- Обращение на «вы» (с маленькой буквы)
- Никаких «осуществляем», «предоставляем», «в рамках»
- Короткие предложения, глаголы в начале: «Заказать», «Выбрать», «Получить»
- Заголовки эмоциональные, не описательные: не «Наши товары», а «Что вы хотите приготовить сегодня?»

## Перед созданием нового файла

Сначала рассказать, что планируется:
- Для кого страница и какую задачу решает
- Какие секции будут
- Есть ли особенности (цвета, контент, структура), которые отличают её от других страниц

Только после подтверждения — приступать к коду.
